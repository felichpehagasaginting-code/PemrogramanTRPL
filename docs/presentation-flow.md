## 1. Overview Presentasi

```
TARGET AUDIENS  : Dosen pembimbing, koordinator divisi, panitia matrikulasi
DURASI          : 25-35 menit (termasuk demo + Q&A)
FORMAT          : Slide + Live Demo + Q&A
SUASANA         : Semi-formal, energetic, forward-looking
TUJUAN UTAMA    : Mendapat approval & dukungan untuk menggunakan platform
```

### 1.1 Struktur Waktu

| Segmen | Durasi | Isi |
|--------|--------|-----|
| Opening + Ice Breaking | 3-5 menit | Menarik perhatian audience |
| Problem Statement | 3-4 menit | Kenapa platform ini perlu ada |
| Solution Overview | 5 menit | Apa yang dibuat |
| Live Demo | 8-10 menit | Show, don't tell |
| Teknis & Keamanan | 3-4 menit | Meyakinkan aspek teknis |
| Roadmap & Tim | 2-3 menit | Timeline dan siapa yang ngerjain |
| Penutup + Call to Action | 2-3 menit | Dorong adopsi |
| Q&A | 5 menit | Tanya jawab |

## 2. Alur Slide yang Disarankan

### Slide 1: Cover
```
Judul    : "Matrikulasi TRPL"
Subjudul : "Belajar Coding: Akhirnya Seru Juga!"
Visual   : Ilustrasi mahasiswa coding dengan ekspresi excited
Footer   : Logo prodi TRPL + tanggal + nama presenter
```

### Slide 2: Ice Breaker – Polling
```
Judul  : "Sebelum mulai, jujur aja ya..."
Konten : Polling real-time (gunakan Mentimeter/Slido)
         - Berapa dari kalian yang pernah ngajar matrikulasi yang boring?
         - Berapa yang maba-nya langsung semangat dari hari pertama?
Visual : QR code ke polling, hasil real-time
```

### Slide 3: Realita Matrikulasi Sekarang
```
Judul  : "Matrikulasi kita hari ini..."
Konten : Split screen:
         KIRI (teks): "Slide PPT tebal, ceramah 2 jam, maba ngantuk"
         KANAN: Meme "Is this a pigeon?" = "Is this engaging?"
Narasi : Datar dan serius, buat audiens relate
```

### Slide 4: Problem Statement
```
Judul  : "Ada 5 masalah yang kita harus selesaikan"
Konten : 5 poin masalah (animasi muncul satu per satu):
         1. Maba takut coding dari awal
         2. Materi satu arah = retensi rendah
         3. Tidak ada cara ukur pemahaman real-time
         4. Tidak ada platform terpusat
         5. Gen-Z gampang bosan
Visual : Icon di tiap poin, warna gradasi dari abu ke merah
```

### Slide 5: Solusi – The Big Reveal
```
Judul  : "Jadi, kami bikin ini:"
Visual : Screenshot/mockup platform yang keren
         (Full screen mockup, tidak ada teks selain nama platform)
Narasi : "Matrikulasi TRPL – platform belajar coding yang anti-boring."
```

### Slide 6: Fitur Utama (1) – Interaktivitas
```
Judul  : "Bukan ceramah. Ini pengalaman."
Konten : 3 card animasi:
         LIVE CODE EDITOR → tulis dan jalankan kode di browser
         KUIS INTERAKTIF  → drag-and-drop, bukan pilgan biasa
         VISUAL LEARNING  → GIF, ilustrasi, bukan teks doang
Visual : Screenshot editor dan kuis
```

### Slide 7: Fitur Utama (2) – Gamifikasi
```
Judul  : "Belajar itu kayak main game. Serius."
Konten : Badge showcase + leaderboard screenshot
         "Setiap modul selesai = poin + badge + konfeti!"
Visual : Animasi badge unlock (jika slide support animasi)
         Leaderboard dengan nama-nama lucu
```

### Slide 8: Fitur Utama (3) – Meme & Vibes
```
Judul  : "Kita paham Gen-Z."
Konten : Contoh meme yang ada di platform:
         "Me naming variables: x, xx, xFinal, xFinalFINAL..."
         Caption resmi: "Kasih nama yang jelas ya!"
Visual : Screenshot meme popup yang kelihatan keren, bukan cringe
Narasi : "Meme bukan gimmick. Ini cara kita bikin materi lebih relatable."
```

### Slide 9: Fitur Utama (4) – Dashboard & Monitoring
```
Judul  : "Panitia juga dapat manfaatnya."
Konten : Screenshot dashboard admin:
         - Progres semua mahasiswa dalam satu layar
         - Siapa yang stuck di modul mana
         - Export laporan satu klik
Visual : Dashboard screenshot yang clean dan informatif
```

### Slide 10: Keamanan – Trust Builder
```
Judul  : "Aman? Tentu. Ini bukan WA group biasa."
Konten : 4 poin keamanan dengan icon perisai:
         Login via SSO kampus (tidak perlu akun baru)
         Enkripsi data end-to-end
         Sandbox terisolasi untuk kode mahasiswa
         Rate limiting + proteksi dari serangan
Visual : Ikon gembok + checkmark di tiap poin
Narasi : Singkat dan meyakinkan, tidak perlu teknis
```

### Slide 11: Stack Teknologi
```
Judul  : "Dibangun dengan teknologi yang proven."
Konten : Logo-logo tech stack dalam grid rapi:
         Next.js | React | Python Sandbox | Supabase
         Vercel  | Redis | Cloudflare    | TypeScript
Narasi : "Bukan teknologi eksperimen. Ini yang dipakai startup top dunia."
```

### Slide 12: Roadmap
```
Judul  : "8 Minggu. Terstruktur. Terukur."
Konten : Timeline visual (gantt chart simpel):
         M1: Planning | M2: Design | M3-4: Core Dev
         M5-6: Full Features | M7: Testing | M8: Launch
Visual : Progress bar warna gradasi
Narasi : "Kami tidak cuma ngomong, ada roadmap yang bisa ditagih."
```

### Slide 13: Tim
```
Judul  : "Siapa yang bikin ini?"
Konten : Foto/avatar dan nama tiap anggota tim
         dengan role masing-masing
         (Frontend, Backend, Design, Content, Security)
Visual : Card dengan border ungu, foto bulat
Narasi : "Tim solid, spesialisasi jelas, komunikasi tight."
```

### Slide 14: Demo – Transisi
```
Judul  : "Nah, cukup ngomong."
Konten : "Sekarang kita lihat langsung."
         [Animasi laptop terbuka]
Visual : Full screen, minimalis
Narasi : Pause dramatis, buka laptop/browser
```

### Slide 15: Penutup – Call to Action
```
Judul  : "Matrikulasi tahun ini, beda."
Konten : 3 poin mengapa perlu adopt sekarang:
         Maba lebih engaged = retensi lebih baik
         Monitoring real-time untuk panitia
         Fondasi digital untuk prodi ke depan
CTA    : "Izinkan kami mengubah matrikulasi TRPL menjadi pengalaman yang diingat seumur hidup."
Visual : Ilustrasi graduation + confetti
```

### Slide 16: Q&A
```
Judul  : "Ada yang mau tanya? Gas!"
Visual : QR code ke platform (staging/preview)
         "Coba sendiri sekarang: [URL]"
```

## 4. Tips Membuka Presentasi

### 4.1 Ice Breaking Terbaik untuk Audiens Ini

**Opsi A: Polling Real-time** (Direkomendasikan)
- Gunakan Mentimeter.com atau Slido
- Pertanyaan: "Seberapa menarik matrikulasi yang kamu ikuti dulu?"
- Skala 1-5, tampilkan hasilnya real-time
- Komentari hasilnya: "Nah, ini masalah yang mau kita selesaikan."

**Opsi B: Tebak Meme**
- Tampilkan meme pemrograman tanpa caption
- Tanya: "Ada yang tau ini maksudnya apa?"
- Segway ke: "Nah, meme kayak gini yang akan ada di platform kita"

**Opsi C: Demo Shock**
- Langsung buka platform tanpa intro
- Minta satu orang dari audiens naik ke depan dan coba fitur coding
- Baru masuk ke penjelasan setelahnya

### 4.2 Kalimat Pembuka yang Powerful

**Versi energetic**:
> "Oke, satu pertanyaan. Siapa yang waktu kuliah dulu pernah ngantuk di matrikulasi? [Lihat respons] Nah, hari ini kita ubah itu."

**Versi storytelling**:
> "Bayangkan kamu mahasiswa baru. Hari pertama. Excited. Masuk ruangan, duduk, dan... dua jam dengerin slideshow. Semangat yang tadi membara langsung padam. Ini yang terjadi setiap tahun. Sampai sekarang."

**Versi direct**:
> "Kami punya solusi untuk membuat maba TRPL tidak cuma hadir di matrikulasi, tapi benar-benar belajar. Dan ini dia."

## 5. Demo Live Website

### 5.1 Rute Demo yang Disarankan (8-10 menit)

```
1. [1 menit] Landing Page
   - Tunjukkan visual yang keren
   - "Ini yang maba lihat pertama kali"

2. [1 menit] Login SSO
   - Demo klik tombol login
   - "Tidak perlu buat akun baru, pakai akun kampus"

3. [2 menit] Dashboard Mahasiswa
   - Tunjukkan progress bar, modul list
   - "Dari sini maba tahu harus ngapain"

4. [3 menit] Modul + Live Code Editor
   - Buka Modul 2 (Variabel)
   - Ketik kode Python sederhana
   - Klik Run, tunjukkan outputnya
   - "Coding langsung di browser, tidak install apapun"

5. [1 menit] Kuis Interaktif
   - Demo kuis pilihan ganda atau drag-and-drop
   - Jawab dengan benar: tunjukkan animasi "Benar!"

6. [1 menit] Badge + Gamifikasi
   - Tunjukkan popup badge setelah selesai
   - "Ini yang bikin maba mau lanjut ke modul berikutnya"

7. [30 detik] Leaderboard
   - Singkat, "ini kompetisi sehat"

8. [30 detik] Dashboard Admin
   - Tunjukkan monitoring progres
   - "Ini untuk panitia, bisa lihat siapa yang butuh bantuan"
```

### 5.2 Persiapan Demo

**Teknis:**
- Test koneksi internet sebelumnya
- Buka tab browser dengan semua halaman yang akan di-demo
- Zoom browser ke 125% agar terbaca dari belakang
- Matikan notifikasi (Do Not Disturb mode)
- Siapkan akun demo (bukan akun asli dengan data sensitif)

**Konten:**
- Seed konten modul sudah ada di platform
- Akun demo mahasiswa dan admin sudah dibuat
- Beberapa kode Python sudah disiapkan (tidak perlu ngetik dari nol)

**Backup Plan:**
- Jika internet mati: siapkan recording video demo
- Jika platform error: siapkan screenshot/mockup Figma
- Jangan panik – katakan "Kami akan tunjukkan versi livenya nanti"

## 6. Menutup Presentasi

### 6.1 Closing Statement yang Kuat

**Versi emosional**:
> "Pengalaman pertama mahasiswa baru dengan coding sangat menentukan. Jika mereka langsung jatuh cinta, mereka akan bertahan. Jika mereka langsung frustrasi, mereka akan struggle sepanjang kuliah.
> Platform ini adalah investasi di hari pertama mereka.
> Kami sudah siap. Tinggal izin dan dukungan dari bapak/ibu."

**Versi impact**:
> "Kami tidak hanya membuat website. Kami membuat 200+ mahasiswa baru tahun ini memiliki pengalaman matrikulasi yang mereka ceritakan ke orang tua, ke teman-teman, bahkan ke calon mahasiswa tahun depan.
> Itu branding TRPL yang tidak bisa dibeli."

**Versi data-driven**:
> "Platform belajar interaktif terbukti meningkatkan retensi materi hingga 60% dibanding metode konvensional. Kami tidak bereksperimen – kami mengimplementasikan apa yang sudah terbukti, dengan konteks TRPL kita."

### 6.2 Call to Action yang Jelas

Selalu akhiri dengan satu permintaan yang spesifik:
> "Yang kami butuhkan dari bapak/ibu hari ini ada tiga:
> 1. Approval untuk gunakan platform ini di matrikulasi tahun ini
> 2. Akses ke endpoint SSO kampus untuk integrasi login
> 3. Satu sesi validasi kurikulum dengan dosen TRPL
> Itu saja. Sisanya kami handle."

## 7. FAQ yang Mungkin Muncul

| Pertanyaan | Jawaban Singkat yang Menjual |
|------------|------------------------------|
| "Apakah ini aman untuk data mahasiswa?" | "Login via SSO kampus yang sudah ada. Data terenkripsi. Tidak ada data baru yang dikumpulkan tanpa perlu." |
| "Bagaimana jika server down saat matrikulasi?" | "Kami jalankan load test untuk 250+ user. Plus ada monitoring 24/7 dengan alert otomatis." |
| "Berapa biaya operasional?" | "Estimasi $60-80/bulan. Jauh lebih murah dari materi cetak yang dibuang setelah matrikulasi." |
| "Apakah mahasiswa yang tidak punya laptop bisa pakai?" | "Desainnya mobile-friendly. Smartphone sudah cukup untuk akses materi dan kuis." |
| "Bagaimana jika koneksi kampus lambat?" | "Aset statis di-cache di CDN global. Materi bisa diakses walau koneksi sedang." |
| "Siapa yang maintain setelah matrikulasi selesai?" | "Tim diwarisi ke divisi IT atau mahasiswa senior TRPL yang bisa jadi maintainer." |
| "Apakah bisa dipakai untuk angkatan berikutnya?" | "Ya! Ini investasi jangka panjang. Konten bisa diupdate tiap tahun." |

## 8. Checklist Sebelum Presentasi

### H-3 (3 Hari Sebelum)
- [ ] Slide final sudah direviw 2x
- [ ] Demo sudah dicoba dari awal sampai akhir
- [ ] Akun demo sudah dibuat dan berfungsi
- [ ] Backup video demo sudah direcord
- [ ] Polling Mentimeter/Slido sudah dibuat

### H-1 (Sehari Sebelum)
- [ ] Test projector/layar di ruang presentasi
- [ ] Test koneksi internet di ruang presentasi
- [ ] Zoom browser ke 125-150% untuk keterbacaan
- [ ] Siapkan backup di USB atau Google Drive
- [ ] Rehearsal lengkap di depan cermin atau rekan

### Hari H (Sebelum Presentasi)
- [ ] Tiba 30 menit lebih awal
- [ ] Connect laptop ke projector + test
- [ ] Buka semua tab yang dibutuhkan
- [ ] Aktifkan Do Not Disturb
- [ ] Tarik napas. Platform kita keren. Percaya diri!
